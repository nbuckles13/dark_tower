## Cross-cutting patterns (ADR-0032 derived)
- **Catalog "expected vs enforced" honesty pattern (Step 5 F1)**: when a Prometheus label is sourced from JWT claims, request bodies, headers, or any externally-supplied unbounded value, the catalog MUST distinguish the *expected/legitimate* value enumeration from what the emission site *enforces*. If the code does not allowlist-clamp before recording, add a "Cardinality note" line adjacent to the existing Cardinality bullet citing (a) the unclamped emission site by file:line, (b) the production trust boundary that bounds severity (e.g., JWKS issuance — only legitimately-issued tokens reach the site), and (c) a `docs/TODO.md` entry tracking the clamping fix with LoC estimate. Reference example: `docs/observability/metrics/gc-service.md` §`gc_caller_type_rejected_total` (Cardinality + Cardinality note bullets, driven by unclamped read at `crates/gc-service/src/grpc/auth_layer.rs:241`).
- **Shared-metric-family discriminator label pattern (Step 5 §`participant`)**: when two handlers produce the same outcome shape (e.g., GC `join_meeting` user-path + `get_guest_token` guest-path both emit join token + MC assignment), reuse one metric family with a discriminator label rather than forking into parallel `*_user_*` / `*_guest_*` families. Splitting forks the SLO and doubles dashboard panels. Operator-differentiation comes via the discriminator label (e.g., `participant`); per-handler error semantics surface via path-specific `error_type` values (e.g., `guests_disabled` exclusive to guest path). Catalog must enumerate label values at the metric, NOT just label names; cardinality bound checked against ADR-0011 cap-10. Reference example: `gc_meeting_join_*` family at `docs/observability/metrics/gc-service.md:200-269`.
- **Label-domain exclusivity invariant testing pattern (Step 5)**: when an `error_type` value is exclusive to one branch of a discriminator-labelled metric (e.g., `guests_disabled` is reachable only from the guest path predicate `!meeting.allow_guests`), pin that exclusivity as a dedicated `assert_unobserved` test on the OTHER discriminator value's slot. The test should drive every legitimate same-discriminator emission to prove non-coincidence. This catches predicate-boundary violations that simple coverage cannot. Reference example: `crates/gc-service/tests/meeting_join_metrics_integration.rs:meeting_join_user_unauthorized_does_not_fire_on_guest_path`, `meeting_join_guest_only_error_types_do_not_fire_on_user_path`, plus the third dedicated `meeting_join_guests_disabled_is_guest_exclusive_label_domain_invariant` test.
- **Gauge zero-fill 4-cell adjacency-coverage matrix pattern (Step 5)**: any zero-filling gauge wrapper (one that explicitly sets `0` for absent label combinations rather than omitting them) requires four distinct test cells to cover its semantics: (1) full happy path with all combinations present and non-zero; (2) partial happy path with `assert_value(0.0)` on missing combinations (NOT `assert_unobserved` — wrapper emits explicitly); (3) empty-counts boundary with all combinations zero-filled; (4) caller-short-circuits-before-wrapper with `assert_unobserved` on the full label space (proves `assert_unobserved` catches a refactor that accidentally always-emits, semantically distinct from cell 2). Reference example: `crates/gc-service/tests/registered_controllers_metrics_integration.rs` for `gc_registered_controllers`.
- **Dashboard-panel-defer-with-TODO discipline (Step 5)**: a metric-test backfill devloop should NOT mix in dashboard work. New panels broken out by a newly-added discriminator label (e.g., `by(participant)`) are operator-experience iterations and belong in a follow-up devloop. Existing aggregation panels (`sum by(status)`, etc.) remain stable across new label additions because Prometheus aggregates over unmentioned labels by default — verify this and document in the deferred TODO entry. Reference example: `docs/TODO.md` §"GC dashboard panel for `gc_meeting_join_*` `participant` axis"; rationale: panel-render failures would otherwise escape `validate-metric-coverage.sh`.
