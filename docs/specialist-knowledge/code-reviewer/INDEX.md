## Review Heuristics
- **Plan-stage commitment fidelity (AC iter-2 lesson; recurred ADR-0032 Step 5 GC, recovered)**: silent partial migration vs plan is a discipline failure, not a low-pri nit. Surface as a single bundled finding citing each named gap with file:line evidence, then offer Option-1-or-Option-2 triage (complete the work, or document the deferral by *naming the friction* — e.g., "attack-vector helpers couple to file-specific claim shapes"). The line-N "flag-and-discuss" off-ramp is legitimate; using it without articulation is what's not. Recovery pattern when implementer accepts: re-load each named gap, ship at full fidelity in batched cleanup pass — works because the original slip is usually time-pressure at edit-pass close, not engineering disagreement.
- **Source-of-truth disagreement is a higher-severity finding than either-doc-alone-wrong**: when handler comments and the observability catalog disagree about a label's semantics (e.g., GC Step 5 had handler claiming `forbidden` was "shared" while catalog correctly said user-only), the cross-reference is the bug — even if one of the two is correct in isolation. Cite both file:lines and require both updated to agree. Future-readers consult whichever is closer to their context; divergence guarantees one will be wrong from their POV.
- **Comment-block tightening — don't overclaim what tests prove**: when a cluster test drives a label cell that production never emits (e.g., guest-side `forbidden` in GC Step 5), the test is "wiring-only" not "production-reachable." Per-cell annotation explaining the structural reason (label-wiring proof + predicate-collapse-refactor catch) earns its place; const-name-driven discrimination (`SHARED_ERROR_TYPES` vs `WIRING_ONLY_ERROR_TYPES`) creates future-classification hazard. Prefer per-cell annotation over const split — documentation lives next to the assertion that depends on it.
- **Pre-review-verification division of labor**: when implementer pre-verifies mechanical watch items (alert-query `without()` pattern audit, call-site count, dashboard query correctness) before formal review, accept the verification + reciprocate by focusing review on judgment-call items (parity-comment fidelity, framing-pattern alignment with siblings, scope-fidelity vs plan, comment-block accuracy). Collapses review-time workload meaningfully and preserves the reviewer-as-fresh-eyes value where it matters.
- **`assert_value(0.0)` vs `assert_unobserved` semantic distinction is load-bearing**: gauge zero-fill (wrapper explicitly emits `set(0.0)` for absent labels) requires `assert_value(0.0)` — a refactor that silently drops the explicit zero-fill loop must fail the test. `assert_unobserved` is for code paths that don't touch the metric (caller short-circuits before update_*). Conflating them gives false-positive coverage. Per-cell adjacency matrix should include BOTH cells distinctly: "passes-on-set-zero" (cell 2) and "passes-on-never-set" (cell 4).
- **Re-look discipline: verify against actual source, not diff summary**: when implementer reports "all findings landed," re-grep the named file:lines in the working tree before flipping verdict. The re-look on ADR-0032 Step 5 GC (3rd verification pass) caught no gaps but stayed within budget because each watch item maps to a single grep — the time investment is small, the false-CLEAR cost is large.
